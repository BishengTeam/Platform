**违反标准**：无（属于代码缺陷）

**说明**：`EmptyState` 组件的 Props 定义为 `title: string`（必填），但以下 5 处传入了不存在的 `text` 属性，导致 `title` 必填字段缺失，会产生 TypeScript 编译错误且组件无法正常显示标题：

**建议**：将 `text={...}` 改为 `title={...}`。

### 1.2 FormInput 组件缺少 `disabled` 属性

**违反标准**：无（属于代码缺陷）

**说明**：`src/components/FormInput/index.tsx` 的 `FormInputProps` 接口未定义 `disabled` 属性，但以下文件使用了该属性：

| 文件 | 行号 |
|------|------|
| `src/pages/mine/profile.tsx` | 49-51（3 处） |

由于 `Input` 组件（Taro）原生支持 `disabled`，但 FormInput 未将其传递给内部 `<Input>`，`disabled` 属性不会生效。

**建议**：在 `FormInputProps` 中增加 `disabled?: boolean`，并在 JSX 中透传给 `<Input>`。

## 四、样式规范（违反标准 6）

### 4.1 静态值内联样式

**文件**：`src/pages/registration/form-renshe.tsx:92`

```tsx
<Text style={{ color: '#FF4D4F', fontSize: '12px' }}>{errors.branch.message}</Text>
```

颜色 `#FF4D4F` 和字号 `12px` 均为静态值，应使用 CSS Module class。

**建议**：定义 `.errorText` class，通过 `className` 引用。

### 4.2 渐变定义在多处重复维护

以下位置各自维护了一份渐变色映射，存在不一致风险：

- `src/styles/_variables.scss`（`$gradient-blue` 等 SCSS 变量）
- `src/app.scss`（`--gradient-blue` 等 CSS 变量）
- `src/components/ZoneBanner/index.tsx`（`GRADIENTS` 常量）

**建议**：将渐变定义统一到 `_variables.scss`，ZoneBanner 改为直接从 CSS 变量读取或通过 SCSS 变量编译。


这是一个通用工具函数，不应定义在页面文件中。
