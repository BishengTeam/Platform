## 一、 问题诊断与痛点分析

### 1. 业务数据断层（对齐后端）

* **缺失重要核心字段**：后端返回了 `organization`（所在单位），但图中却没有显示用户的 `school`（毕业院校/在读学校）、`major`（所学专业）以及针对境外用户的 `country` / `pinyin` 等字段。
* **状态语义不一致**：图1的认证状态显示为 `pending`，图2顶部却写着 `认证状态: 审核中`，中英文混杂，应该在前端做统一的映射字典映射。
* **脱敏与非脱敏混淆**：后端细心地返回了 `phone`/`phone_raw` 和 `id_card`/`id_card_raw`。编辑状态下必须回显或允许输入完整的 `_raw` 数据，而不是把带星号的字符串直接存回数据库。

### 2. 视觉与体验痛点（太难看）

* **排版死板**：纯白底色加几根毫无细节的通栏细分割线，像一个粗糙的表格表单，没有移动端（微信小程序）的精致感。
* **头部刘海屏未适配**：顶部胶囊按钮和系统状态栏把页面标题 `个人资料` / `编辑资料` 挤变形了，直接撞到了 iPhone 的系统灵动岛/刘海区域。
* **编辑状态不合理**：点击“编辑资料”后，居然是在原页面直接把展示项变成了输入框，甚至连“保存/提交”按钮都滑到屏幕下面去了，操作极不直观。
* **修改次数限制未突出**：图2中带有 `修改次数: 1/3` 的重要限制提示，直接用暗灰色小字缩在角落，用户极易忽略。

## 三、 代码具体改动落地指引

根据你的项目文件路径，你需要对以下几个文件进行彻底重构：

### 1. 补齐路由与页面跳转 (`src/constants/routes.ts` & `profile.tsx`)

停止在 `profile.tsx` 页面原地切换编辑状态。

* **`profile.tsx`**：只负责纯文本、卡片化、高颜值的**展示**。
* **`edit-profile.tsx`**：点击编辑后，直接路由跳转到此页面，提供标准的表单输入框，底部固定放置“保存修改”原生大按钮。

### 2. 完善国际化与状态字符串字典 (`src/constants/strings.ts`)

在文件中补齐后端状态码的翻译映射，拒绝在前端直接渲染 `pending`：

```typescript
export const IDENTITY_STATUS_MAP = {
  'pending': '审核中',
  'approved': '已认证',
  'rejected': '认证失败',
};

export const MINE_PROFILE_LABELS = {
  SCHOOL: '毕业院校',
  MAJOR: '所学专业',
  AGE: '年龄',
  COUNTRY: '国籍/地区',
};

```

### 3. 数据层改造 (`src/services/userService.ts` & 页面)

处理好脱敏与非脱敏字段的交接：

* 在 `profile.tsx` (展示页) 调用 `getUserProfile`，直接渲染 `phone` 和 `id_card`（带星号的）。
* 在 `edit-profile.tsx` (编辑页) 回显数据时，必须把 `phone_raw` 和 `id_card_raw` 塞进 `input` 的 `value` 中，否则用户一旦点击保存，带星号的错误数据就会把数据库里的正确数据覆盖。

## 四、 大改实施步骤建议

1. **第一步：改 `routes.ts**`。确立“展示页”与“编辑页”彻底分家的路线，拒绝原地切换。
2. **第二步：重构 `profile.tsx` 和它的 scss**。把 `school`、`major`、`age` 等后端有返回的字段用卡片（Card）的结构全部排进去，引入自定义 NavigationBar 避开微信顶部胶囊区。
3. **第三步：重构 `edit-profile.tsx**`。做成真正的表单页，把 `修改次数: 1/3` 放到页面最顶部做 Banner 提示，使用 `*_raw` 字段支持编辑，底部做成吸底的 Form Submit 按钮。
