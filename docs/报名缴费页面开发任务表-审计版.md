# 报名缴费页面开发任务表（审计版）

> **审计日期**: 2026-05-08
> **原始文档**: `docs/报名缴费页面开发任务表.md`，基于《小程序前端页面设计文档 V1》第 3 章节
> **审计结论**: 原始文档基于**原生微信小程序（WXML/WXSS/JS）** 编写，当前项目为 **Taro 4 + React 18 + TypeScript + SCSS Modules**，需要全面转换。项目中已有大量可复用基础设施，但报名/缴费功能本身未开始。

---

## 一、审计摘要

### 1.1 原始文档的主要问题

| 问题 | 说明 |
|------|------|
| **平台不匹配** | 原始文档使用 WXML/WXSS/JS/`app.json`/`wx.request`，当前项目是 Taro + React + TSX + SCSS + `Taro.request` |
| **前置任务重复** | P0（项目骨架）已完成；P1（API 服务层）已有 `dataService.ts` 模式但需扩展；P2（校验工具）需新建 |
| **组件未复用** | 原文档列出 FormInput/PriceRow/EmptyState 作为新建组件，未考虑复用已有的 PageHeader/Button/ZoneCard/TagFilter/AuthGuard |
| **并行策略过时** | 原文档假设两人并行开发，实际为单人开发，应改为顺序执行 |
| **路径命名不符合项目规范** | 原文档用 `components/form-input`、`utils/api.js` 等路径；当前项目用 `src/components/FormInput/`、`src/services/` 等 |
| **缺少关键注册步骤** | 原文档未提及路由注册（`app.config.ts` 的 subPackages）、类型定义（`src/types/`）、字符串常量（`STRINGS`）、mock 数据创建 |

## 二、页面规划

| 页面 | 路由 | 核心功能 | 依赖页面 |
|------|------|----------|----------|
| 报名专区 | `pages/registration/index` | 搜索栏 + 分类Tab + 认证卡片列表 | 无 |
| 报名表单 | `pages/registration/form` | 表单校验 + 考试券条件显隐 + 费用明细 | `index`（传递 cert 参数） |
| 报名确认 | `pages/registration/confirm` | 订单汇总 + 微信支付调用 + 协议勾选 | `form`（传递表单数据） |
| 支付结果 | `pages/payment/result` | 成功/失败双状态展示 | `confirm`（传递支付结果） |

**数据传递方式**: 页面间通过 Taro 路由参数（`Taro.navigateTo({ params })`）传递关键标识（如 `cert_id`、`order_id`），完整表单数据通过 `Taro.setStorageSync` / `Taro.getStorageSync` 暂存。

## 六、API 接口对照

| 页面 | 接口 | 方法 | 请求参数 | 响应 |
|------|------|------|----------|------|
| 报名专区 | `/api/certifications` | GET | `?category=` (可选) | `Certification[]` |
| 报名确认 | `/api/orders` | POST | `{ cert_id, real_name, phone, id_card, identity_type, coupon_code? }` | `{ order_id, amount }` |
| 报名确认 | `/api/payment/prepay` | POST | `{ order_id }` | `{ timeStamp, nonceStr, package, signType, paySign }` |
| 支付结果 | `/api/orders/{id}` | GET | — | `{ id, status, amount, ... }` |

## 七、与现有功能的对接点

| 对接点 | 说明 |
|--------|------|
| 专区入口 | `ZonesContent` 中添加"报名专区"入口卡片，或考试/竞赛专区的 ZoneCard 的"立即报名"按钮跳转报名表单 |
| 首页 AI 对话 | `ChatArea` 中考试卡片可携带"报名"按钮，跳转报名专区 |
| 我的订单 | 现有 `pages/orders/` 已展示订单列表，支付成功后的订单会出现在此处，状态从"待付款"→"已报名" |
| 消息通知 | 报名/支付成功后 mock 一条通知到 `pages/notifications/` |
| 登录鉴权 | 所有报名/支付页面外层包裹 `AuthGuard`（或复用 `ZonePage` 的内置鉴权） |
