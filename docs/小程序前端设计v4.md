| **默认回复** | 其他 | 引导文本，提示用户访问专区或继续提问 |

**智小通知知识库覆盖范围**：
- 考试信息查询 / 科目介绍
- 科目推荐（根据用户身份/年龄推荐）
- 费用咨询
- 研学活动 / 研学报名咨询

**对话策略**：先提问用户（身份类型、年龄段等），收集信息后再做个性化推荐，也支持用户直接自主提问。

### 2.3 核心组件

#### 2.3.1 消息区域 `ChatArea`

渲染对话消息列表，支持自动滚动到底部。

```
 props:
   - messages: Message[]         // 消息列表
   - isTyping: boolean           // 是否正在等待 AI 回复
   - onCardTap: (zoneKey) => void // 卡片点击回调
```

**Message 类型**：

| 字段 | 类型 | 说明 |
|------|------|------|
| id | number | 消息唯一标识 |
| type | 'user' \| 'ai' | 消息发送方 |
| text | string | 消息文本内容 |
| card | ChatCard? | 可选的卡片数据 |

**ChatCard 类型**：

| card.type | 说明 | 携带字段 |
|-----------|------|----------|
| `teacher` | 讲师联系卡片 | name, title, wechat, phone |
| `exam` | 考试信息卡片 | title, description, price, tag |
| `zone_link` | 专区链接卡片 | zoneName, zoneKey, description |
| `course` | 课程推荐卡片 | title, description, duration, tag |

#### 2.3.2 输入区 `ChatInput`

```
 props:
   - value: string
   - shouldShowQuickQuestions: boolean
   - quickQuestions: string[]
   - onInput: (value) => void
   - onSend: () => void
   - onQuickTap: (question) => void
```

- 文本输入框 + 发送按钮
- 上方快捷问题栏：横向 `scroll-view`，点击快捷问题直接发送
- 默认快捷问题由 `getQuickQuestions()` 提供，后台可配置

#### 2.3.3 悬浮客服按钮 `FloatingService`

```
┌────┐
│ 🎧 │  ← 可拖拽悬浮按钮
└────┘
```

- **位置**：页面右侧悬浮，起始 `top: 300px`，可上下拖拽
- **拖拽吸附**：松手时根据触摸位置自动吸附到左侧或右侧边缘（与边缘距离 24rpx）
- **点击判断**：移动距离 < 6px 视为点击，否则为拖拽
- **图标**：headset 耳机图标（`#1677FF` 科技蓝）
- **点击**：`navigateTo` 到人工客服页 `pages/service/index`

#### 2.3.4 打字状态指示 `TypingIndicator`

AI 思考中显示"正在输入..."动画（三个跳动圆点），通过 `isTyping` 状态控制。

### 2.4 交互流程

```
用户输入问题 / 点击快捷问题
    │
    ▼
前端展示用户消息气泡（optimistic UI）
    │
    ▼
显示 typing 状态（"正在输入..."动画）
    │
    ▼
根据关键词匹配意图:
  - "老师/讲师/联系"    → teacher 卡片（讲师联系方式）
  - "考试/报名/认证/H3C" → exam 卡片（考试信息 + 文本说明）
  - "课程/学习/培训"     → course 卡片（课程推荐）
  - "竞赛/比赛"         → zone_link 卡片（跳转竞赛专区）
  - "活动/研学"         → zone_link 卡片（跳转活动专区）
  - "就业/工作/招聘"     → zone_link 卡片（跳转就业专区）
  - 无匹配              → 默认引导文本
    │
    ▼
用户点击卡片按钮 → 跳转对应页面
```

### 2.5 状态管理

AI 咨询页使用页面级 `useState` 管理状态：

```javascript
// pages/ai-consult/index.tsx
const [messages, setMessages] = useState<Message[]>(初始欢迎消息)
const [inputValue, setInputValue] = useState('')
const [isTyping, setIsTyping] = useState(false)
```

- `messages`：当前会话消息列表（含用户消息和 AI 回复）
- `inputValue`：输入框文本
- `isTyping`：是否正在等待 AI 回复（控制 typing 动画）
- 无全局会话状态，每次进入为独立会话

## 3. 认证专区与报名流程（入口页 `pages/registration/index`）

### 3.1 认证分类

| 认证类别 | 说明 |
|----------|------|
| **H3C 认证** | H3CNE / H3CSE / H3CIE-RS+ 等所有科目，考试代码与中文全称自动匹配 |

**表单校验规则**：
- 真实姓名：必填，2-20 字符
- FirstName / LastName：必填（与厂商报名表对齐）
- 手机号：必填，正则校验
- 身份证号：必填，18 位格式校验
- 身份证照片：JPG 格式，小于 5MB
- 电子邮件：邮箱格式校验

## 15. 状态管理与数据流

使用页面级 `useState` 管理状态，跨页面传递使用 URL 参数或本地存储。

```javascript
// hooks/useAuth.ts
// 认证状态管理，通过 AuthGuard 组件包裹需登录页面
// 页面级使用 useState + Taro.getStorageSync/setStorageSync 传递数据
```

- **认证状态**：`AuthGuard` 组件（hooks/useAuth.ts），包裹所有需登录页面
- **表单数据传递**：`registration/form → registration/confirm` 通过 `Taro.setStorageSync('registration_form_data', ...)` 传递
- **页面参数传递**：通过 URL query 参数（如 `?cert_id=xxx`、`?order_id=xxx&status=xxx`）
- **AI 对话状态**：页面级 `useState`，每次进入为独立会话

## 16. 性能与体验优化

1. **图片懒加载**：专区列表、商品卡片使用 `lazy-load`
2. **列表分页**：订单、专区内容触底分页加载
3. **输入防抖**：AI 对话输入框 300ms 防抖
4. **本地缓存**：用户资料、认证列表、常见问题 `wx.setStorageSync`
5. **分包加载**：五大专区作为独立分包
6. **首屏优化**：首页骨架屏 + 推荐问题预加载
