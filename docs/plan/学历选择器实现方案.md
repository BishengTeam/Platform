# 报名页学历选择器实现方案

> 创建时间：2026-06-16
> 决策：**纯前端实现**（理由见下文）

---

## 一、当前状态

学历字段在报名表单中是通过 `FormInput` 组件渲染的普通文本 `<Input>`，用户需手动打字输入。但 placeholder 已经写的是"请选择学历"，说明设计意图从一开始就是"选择"而非"自由输入"。

涉及两个位置：

| 文件 | 行号 | 代码 |
|------|------|------|
| `src/pages/registration/components/H3CExtraSection.tsx` | 约 L72 | `<FormInput label={STRINGS.FORM_EDUCATION} placeholder={STRINGS.FORM_EDUCATION_PLACEHOLDER} value={education} onChange={setEducation} />` |
| `src/pages/registration/components/NispExtraSection.tsx` | 约 L82 | `<FormInput label={STRINGS.FORM_EDUCATION} placeholder={STRINGS.FORM_EDUCATION_PLACEHOLDER} value={education} onChange={setEducation} />` |

Native 已有的常量：

```typescript
// src/constants/strings.ts L380-381
FORM_EDUCATION: '学历',
FORM_EDUCATION_PLACEHOLDER: '请选择学历',
```

## 二、为什么是前端实现

| 维度 | 前端 | 后端 |
|------|------|------|
| **数据性质** | 学历是固定枚举（博士/硕士/本科/大专/高中/初中及以下），十年不变 | 需要后端定义接口、存配置、维护字典表 |
| **响应速度** | 点击即弹 Picker，零延迟 | 需 `Taro.request` → 等网络 → 再弹选择器，弱网下体验崩塌 |
| **离线可用** | ✅ | ❌ |
| **变更成本** | 学历加一档 → 改一行前端常量，发版即可 | 学历加一档 → 改后端配置/数据库 → 改接口文档 → 前端对接新字段 |
| **项目一致性** | 性别、身份类型、国家、语言、NISP 级别等所有枚举型字段全是前端按钮组/Picker | 没有任何类似字段走后端 |
| **耦合度** | 后端只收字符串 `"本科"`，不关心选项长什么样 | 后端需同时维护枚举值和展示逻辑 |

结论：学历选择没有一丝理由走后端。

## 三、技术方案

### 3.1 总体思路

微信小程序原生提供 `<picker mode='selector'>` 组件，Taro 中直接可用。新建一个 `FormPicker` 组件封装它，外观与 `FormInput` 保持一致，但点击后弹出系统原生滚轮选择器。

### 3.2 文件变更清单

| 操作 | 文件 | 说明 |
|------|------|------|
| **新建** | `src/components/FormPicker/index.tsx` | 封装 Picker 的表单选择组件 |
| **新建** | `src/components/FormPicker/index.module.scss` | 组件样式（复用 FormInput 的视觉效果） |
| **修改** | `src/constants/strings.ts` | 新增 `EDUCATION_OPTIONS` 常量 |
| **修改** | `src/pages/registration/components/H3CExtraSection.tsx` | 学历字段：`FormInput` → `FormPicker` |
| **修改** | `src/pages/registration/components/NispExtraSection.tsx` | 学历字段：`FormInput` → `FormPicker` |

不涉及后端任何改动。

### 3.3 组件设计 — `FormPicker`

组件接口：

```typescript
interface FormPickerProps {
  label: string
  value: string
  options: string[]          // 选项列表
  placeholder?: string       // 未选择时显示的文字
  error?: ValidationResult
  onChange: (value: string) => void
}
```

核心逻辑：
- 点击区域触发 `<Picker mode='selector' range={options}>`
- `onChange` 事件中从 `e.detail.value` 取出选中索引，映射到 `options[index]` 回传
- 未选择时显示 placeholder（灰色），已选择时显示选中值（深色）
- 错误态与 `FormInput` 一致：红色边框 + 错误提示文字

### 3.4 学历选项常量

```typescript
// src/constants/strings.ts 新增
EDUCATION_OPTIONS: [
  '博士',
  '硕士',
  '本科',
  '大专',
  '高中/中专',
  '初中及以下',
] as const,
```

### 3.5 替换点

**H3CExtraSection.tsx**：

```tsx
// 修改前
<FormInput
  label={STRINGS.FORM_EDUCATION}
  placeholder={STRINGS.FORM_EDUCATION_PLACEHOLDER}
  value={education}
  onChange={setEducation}
/>

// 修改后
<FormPicker
  label={STRINGS.FORM_EDUCATION}
  placeholder={STRINGS.FORM_EDUCATION_PLACEHOLDER}
  value={education}
  options={STRINGS.EDUCATION_OPTIONS}
  onChange={setEducation}

```

**NispExtraSection.tsx**：同理，改动完全一致。

### 3.6 样式策略

`FormPicker` 的外观应完全对齐 `FormInput`：
- Label 行（含红色 `*` 必填标记）：复用 `FormInput` 的 `labelRow` / `asterisk` / `label` 样式
- 选择区域：与 `FormInput` 的 `.input` 相同的宽高（`100% × 80px`）、背景（`$color-bg-tertiary`）、圆角（`$radius-sm`）、边框逻辑
- 选中值的文字样式：与 Input 的 value 相同（`$font-base` / `$color-text`）
- Placeholder 样式：`$color-text-tertiary`
- 右侧加一个向下的箭头图标，暗示可点击展开

## 四、组件伪代码

```tsx
// src/components/FormPicker/index.tsx

import { View, Text, Picker } from '@tarojs/components'
import type { ValidationResult } from '@/utils/validator'
import styles from './index.module.scss'

interface FormPickerProps {
  label: string
  value: string
  options: readonly string[]
  placeholder?: string
  error?: ValidationResult
  onChange: (v: string) => void
}

export function FormPicker({
  label,
  value,
  options,
  placeholder = '请选择',
  error,
  onChange,
}: FormPickerProps) {
  const selectedIndex = options.indexOf(value)
  const showError = error && !error.valid
  const displayText = value || placeholder

  return (
    <View className={styles.wrap}>
      <View className={styles.labelRow}>
        <Text className={styles.label}>{label}</Text>
      </View>
      <Picker
        mode='selector'
        range={options as string[]}
        value={selectedIndex >= 0 ? selectedIndex : 0}
        onChange={e => {
          const idx = e.detail.value as number
          onChange(options[idx])
        }}
      >
        <View className={`${styles.picker} ${showError ? styles.pickerError : ''}`}>
          <Text className={value ? styles.value : styles.placeholder}>
            {displayText}
          </Text>
          <Text className={styles.arrow}>▼</Text>
        </View>
      </Picker>
      {showError && <Text className={styles.error}>{error.message}</Text>}
    </View>
  )
}
```

## 六、风险评估

- **风险**：无。`Picker` 是微信小程序原生组件，Taro 完全支持，兼容性无问题。
- **向后兼容**：提交的数据不变（仍是 `education: "本科"` 字符串），后端无需任何改动。
- **回滚**：若出现问题，把两处 `FormPicker` 改回 `FormInput` 即可，一行 diff。
