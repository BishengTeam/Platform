> 基准文档：`docs/用户信息架构重构方案.md`

## 一、后端变更速览

| 变更 | 说明 |
|------|------|
| `user_identity` 拆分 | 拆为 `user_profile`(L1) + `user_realname`(L2) + `user_student`(L2) + `user_enterprise`(L2) |
| `GET /api/user/profile` | 返回结构从平铺 → 嵌套聚合 `{ profile, realname, student?, enterprise? }` |
| `PUT /api/user/profile` | 仅接受 L1 字段：`nickname, email, phone` |
| 新增 `PUT /api/user/identity` | 修改实名信息（触发审核） |
| 新增 `POST /api/user/student` | 首次提交学生信息（触发审核） |
| 新增 `PUT /api/user/student` | 修改学生信息（触发审核） |
| 新增 `POST /api/user/enterprise` | 首次提交企业信息（触发审核） |
| 新增 `PUT /api/user/enterprise` | 修改企业信息（触发审核） |
| `edit_count` → `level2_edit_count` | 全局 5 次 / 30 天，位于 `user` 聚合根 |
| `gender`/`age` | 后端从身份证自动计算，只读 |
| pending 读取策略 | 用户立即看到新值；驳回时恢复旧值（快照） |

## 二、改动清单

### 2.1 Service 层 — `src/services/userService.ts`

#### 2.1.1 `getUserProfile` 响应类型重构

```
旧 (平铺):
{
  real_name, user_type, gender, phone, email, id_card,
  identity_status, education, school, major, organization,
  id_card_raw, phone_raw, pinyin, first_name, last_name, age
}

新 (嵌套):
{
  // Level 1
  profile:     { nickname, email, phone },
  // Level 2
  realname:    { user_type, real_name, id_card_number, gender, age, status, ... },
  student?:    { education, school, major, student_card_oss, status, ... },
  enterprise?: { organization, status, ... },
  // 全局
  level2_edit_count: number,
  level2_edit_reset:  string,  // ISO timestamp
}
```

**动作**：新增 `UserProfileAggregated` 类型，旧字段映射到新路径。

#### 2.1.2 `updateUserProfile` 参数缩减

```
旧: { phone, email, gender, education, school?, major?, organization? }
新: { nickname?, email?, phone? }
```

**动作**：修改函数签名，类型声明对齐。

#### 2.1.3 新增 4 个 Service 函数

| 函数 | 方法 | 路径 |
|------|------|------|
| `updateIdentity()` | PUT | `/api/user/identity` |
| `submitStudent()` | POST | `/api/user/student` |
| `updateStudent()` | PUT | `/api/user/student` |
| `submitEnterprise()` | POST | `/api/user/enterprise` |
| `updateEnterprise()` | PUT | `/api/user/enterprise` |

**动作**：在 `userService.ts` 或 `authService.ts` 中新增。

## 三、风险点

| # | 风险 | 影响 | 缓解 |
|---|------|------|------|
| 1 | `id_card_raw` 路径变化 | `IdentityCheckGate` 无法获取明文身份证自动提交 | 确认后端在新结构中是否保留该字段及路径 |
| 2 | `phone_raw` 路径变化 | 报名表单手机号自动填表可能失败 | 同上 |
| 3 | `pinyin`/`first_name`/`last_name` 是否保留 | 报名表单拼音字段需手动计算 | 确认后端是否继续返回 |
| 4 | `user_type` 位置变化 | `isStudent` 判断依据路径改变 | 从 `realname.user_type` 读取 |
| 5 | 学生/企业子表可能为 `null` | 渲染前需判空 | TypeScript 可选链 + 条件渲染 |
