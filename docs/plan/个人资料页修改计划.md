## 一、当前状态

### 1.1 后端接口实际返回

```
GET /api/user/profile

{
  real_name:        string      — '张三'
  user_type:        string      — 'student' | 'enterprise' | 'social'
  gender:           string      — 'male' | 'female'
  phone:            string      — '138****1234'  (已脱敏)
  email:            string      — 'zhangsan@example.com'
  id_card:          string      — '1101**********1234'  (脱敏)
  identity_status:  string      — 'pending' | 'verified' | 'rejected'
  education:        string      — '本科'
  school:           string      — '清华大学'
  major:            string      — '计算机科学与技术'
  organization:     string      — '新华三集团'
}
```

`PUT /api/user/profile` 接受的字段：
```
phone, email, gender, education, school, major, organization
```

### 1.2 前端现有实现

| 文件 | 功能 | 问题 |
|------|------|------|
| `personal-info.tsx` | 纯展示，列表式排列所有字段 | 无分组、无状态标签样式、认证状态直接渲染后端原始值 |
| `edit-profile.tsx` | 表单编辑，修改次数限制 3 次 | 修改次数提示不醒目、未处理脱敏字段回写 |
| `userService.ts` | `getUserProfile` / `updateUserProfile` | 正常 |

### 1.3 当前路由

```
/pages/mine/profile        → 设置菜单页（包含「个人资料」入口）
/pages/mine/personal-info  → 资料展示页
/pages/mine/edit-profile   → 资料编辑页
```

展示与编辑已分离为独立页面，符合建议文档要求。

## 二、改动计划

### 2.1 状态映射字典 — `src/constants/strings.ts`

**现状**：认证状态/性别等后端返回英文原始值，前端直接渲染。

**改为**：新增统一映射字典，展示和编辑页共用。

```ts
// 新增常量
export const IDENTITY_STATUS_MAP: Record<string, string> = {
  pending:  '审核中',
  verified: '已认证',
  rejected: '未通过',
}

export const GENDER_MAP: Record<string, string> = {
  male:   '男',
  female: '女',
}

export const USER_TYPE_MAP: Record<string, string> = {
  student:    '学生',
  enterprise: '企业',
  social:     '社会人士',
}
```

**影响文件**：`strings.ts`、`personal-info.tsx`、`edit-profile.tsx`

---

### 2.2 展示页卡片化重构 — `src/pages/mine/personal-info.tsx`

**现状**：单一 `infoRows` 数组，所有字段平铺，通栏分割线。

**改为**：拆分为三个逻辑卡片，引入状态标签组件。

```
卡片 A：核心身份
  └── 昵称、真实姓名、性别、年龄（如后端返回）

卡片 B：认证与安全
  └── 认证状态（彩色标签）、身份证号、手机号、邮箱

卡片 C：组织背景
  └── 按 user_type 分支：
      student → 学校、专业、学历
      其他    → 单位、学历
```

**样式改动**：
- 卡片化设计：白色卡片 + 圆角 16px + 浅阴影
- 认证状态使用彩色标签（pending 橙色、verified 绿色、rejected 红色）
- 底部「编辑资料」按钮保持

**影响文件**：`personal-info.tsx`、`personal-info.module.scss`

---

### 2.3 编辑页脱敏字段处理 — `src/pages/mine/edit-profile.tsx`

**现状**：
- 手机号直接取 `profile.phone`（已脱敏 `138****1234`），用户编辑后可能把星号存回数据库
- 修改次数提示仅为灰色小字

**改为**：

#### 2.3.1 手机号脱敏处理

```
方案 A（推荐）：手机号字段禁止编辑，仅展示脱敏值
  说明：手机号通过微信授权获取，不应手动编辑

方案 B：后端提供 phone_raw 字段
  编辑页用 phone_raw 回显，保存时传原始值
  如后端未提供，则采用方案 A
```

#### 2.3.2 修改次数提示醒目化

将 `修改次数: 1/3` 从灰色小字改为 Banner 样式：
- 浅蓝色背景条 + 图标
- 置于表单顶部

**代码改动**：
```tsx
// 当前：灰色小字
<Text className={styles.editCountText}>
  {STRINGS.MINE_PROFILE_EDIT_COUNT}: {remaining}/{MAX_EDITS}
</Text>

// 改为：Banner 样式
<View className={styles.quotaBanner}>
  <Icon name='info' size={20} color='#1677FF' />
  <Text className={styles.quotaText}>
    本月还可修改 {remaining} 次（共 {MAX_EDITS} 次）
  </Text>
</View>
```

**影响文件**：`edit-profile.tsx`、`edit-profile.module.scss`

---

---

### 2.5 `updateUserProfile` 接口适配

**现状**：接受 `phone, email, gender, education, school, major, organization`

**改为**：
- 如手机号采用方案 A（禁止编辑），`phone` 从 `updateUserProfile` 调用中移除
- 其余字段不变

**影响文件**：`edit-profile.tsx`（handleSave 函数）、`userService.ts`（类型声明）

- `profile.tsx`（设置菜单页）— 不在本次范围内
