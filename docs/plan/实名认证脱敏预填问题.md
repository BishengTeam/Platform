# 实名认证脱敏数据预填问题报告

> 日期：2026-06-09
> 来源：接口测试发现

---

## 问题描述

进入认证报名信息填写页面时，`POST /api/user/identity` 返回 `422`：

```json
{ "code": 40001, "message": "身份证号前 17 位必须为数字" }
```

## 根因

前端预填逻辑链存在脱敏数据泄漏：

```
1. 页面加载 → GET /api/user/identity
   返回: { "id_card_number": "1101**********1237" }   ← 脱敏

2. 前端预填到表单 idCard 字段

3. 用户提交 → POST /api/user/identity
   body: { "id_card_number": "1101**********1237" }   ← 含 * 号
   
4. validate_id_card() → 前 17 位含非数字字符 → 422
```

## 修复方向

**前端**：预填时区分数据来源

| 数据 | 正确来源 | 错误来源 |
|------|---------|---------|
| 姓名 | `GET /api/user/profile` → `real_name` | — |
| 手机号 | 微信授权 `getPhoneNumber` 或 `profile.phone_raw` | `profile.phone`（脱敏） |
| 身份证号 | `profile.id_card_raw`（需已实名）或用户手动输入 | `GET /api/user/identity` → `id_card_number`（脱敏） |

**后端**：`GET /api/user/identity` 返回的 `id_card_number` 已标注脱敏（前 4 + 后 4），前端不应将其作为可提交值。

## 影响范围

- 已有实名认证信息但 `status="rejected"` 的用户 — 前端预填脱敏数据 → 提交必失败
- 首次认证用户 — 不受影响（无实名记录，不会预填）

## 修复建议

1. 前端预填身份证明文时，改用 `GET /api/user/profile` 的 `id_card_raw` 字段
2. 后端在 `POST /api/user/identity` 中增加明确错误提示：`"请填写完整身份证号，不可使用脱敏数据"`
