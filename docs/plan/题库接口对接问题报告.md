## 二、🔴 致命问题

### 2.1 错题本列表 / 收藏列表：响应格式解析错误

**影响页面**：错题本 (`pages/quiz/wrong-book`)、收藏 (`pages/quiz/collections`)

**根因**：后端返回的是 `QuizRecordQuestionResponse` 结构，题目数据嵌套在 `question` 字段中。前端用 `toQuizQuestion()` 直接从顶层读取，拿到的全部是错误字段。

**后端响应结构**（`record_question_payload` 构造）：

```json
{
  "items": [
    {
      "id": 1,
      "question_id": 5,
      "user_answer": "A",
      "is_correct": true,
      "is_wrong": false,
      "is_collected": true,
      "question": {                    // ← 题目数据在这里
        "id": 5,
        "category_id": 3,
        "question_type": "single_choice",
        "question_text": "以下关于OSPF协议的说法，正确的是？",
        "options": { "A": "使用TCP传输", "B": "距离矢量协议", "C": "链路状态算法", "D": "不支持区域划分" },
        "explanation": "OSPF是链路状态路由协议"
      }
    }
  ]
}
```

**前端当前代码**（`quizService.ts` 第 72~89 行）：

```typescript
export async function getWrongBook() {
  const res = await get<any>('/api/quiz/wrong-book')
  const data = res.data as any
  const items: any[] = data?.items || data || []
  return items.map(toQuizQuestion)  // ← 传的是 record 对象，不是 question 对象
}

export async function getFavoriteQuestions() {
  const res = await get<any>('/api/quiz/collections')
  const data = res.data as any
  const items: any[] = data?.items || data || []
  return items.map(toQuizQuestion)  // ← 同上
}
```

**`toQuizQuestion` 读取的字段 vs 实际位置**：

| `toQuizQuestion` 读取 | 实际在后端响应中的位置 | 拿到的值 |
|----------------------|---------------------|---------|
| `raw.id` | `raw.id`（record ID） | **record ID**，非 question ID |
| `raw.category_id` | `raw.question.category_id` | `undefined` → `""` |
| `raw.question_text` | `raw.question.question_text` | `undefined` → `""` |
| `raw.options` | `raw.question.options` | `undefined` → `[]` |
| `raw.question_type` | `raw.question.question_type` | `undefined` → 默认 `"single"` |

**后果**：错题本和收藏页展示的题目内容全部为空。

**修复**：

```typescript
// quizService.ts
return items.map(item => toQuizQuestion(item.question))  // 读嵌套的 question 字段
```

### 2.2 收藏功能：调用了错误的 API

**影响页面**：题库练习页 (`pages/quiz/practice`) 的收藏按钮、收藏页 (`pages/quiz/collections`) 的取消收藏

**根因**：前端 `addFavorite` / `removeFavorite` 来自 `userService.ts`（通用收藏），调用的是 `/api/collections`。但后端题库收藏的正确端点是 `/api/quiz/collections`。

**前端当前调用**（`userService.ts` 第 239~247 行，由 `dataService.ts` 重导出）：

```typescript
// 这些函数本意是通用收藏（课程等），但被题库页面误用了
export async function addFavorite(courseId: number): Promise<void> {
  await post(`/api/collections`, { target_type: 'course', target_id: courseId })
  //     ^^^^^^^^^^^^^^^^ 通用收藏 API        ^^^^^^^^^^^^ 按课程类型传参
}

export async function removeFavorite(courseId: number): Promise<void> {
  await del(`/api/collections`, { target_type: 'course', target_id: courseId })
}
```

**后端正确的题库收藏 API**（`Backend/app/api/quiz.py`）：

```
POST   /api/quiz/collections       body: { question_id: int }
DELETE /api/quiz/collections/{id}   path: record ID
```

**参数语义差异**：

| | 前端当前 | 后端期望 |
|---|---------|---------|
| URL | `/api/collections` | `/api/quiz/collections` |
| 请求体 | `{ target_type: 'course', target_id }` | `{ question_id }` |
| 删除路径 | `DELETE /api/collections` + body | `DELETE /api/quiz/collections/{id}` |

**修复**：在 `quizService.ts` 中新增题库专用收藏函数，替代 `userService.ts` 的通用版本：

```typescript
// quizService.ts 新增
export async function addQuizFavorite(questionId: number): Promise<void> {
  if (USE_MOCK) return
  await post('/api/quiz/collections', { question_id: questionId })
}

export async function removeQuizFavorite(id: number): Promise<void> {
  if (USE_MOCK) return
  await del(`/api/quiz/collections/${id}`)
}
```

然后在 `dataService.ts` 中将 `addFavorite`/`removeFavorite` 的导出源从 `userService` 改为 `quizService`，或新增导出名。

## 三、⚠️ 中等问题

### 3.1 答案提交：选项下标 vs 选项标签

**影响页面**：练习页 (`pages/quiz/practice`)、模拟考试页 (`pages/quiz/mock`)

**根因**：前端提交答案时传的是选项数组下标（`"0"`, `"1"`, `"2"`），但后端数据库中 `correct_answer` 存储的是选项标签（`"A"`, `"B"`, `"C"`），导致判分永远失败。

**前端当前代码**（`practice.tsx` 第 41 行）：

```typescript
const handleSelectSingle = useCallback((questionId: string, optIndex: number) => {
  setAnswers(prev => ({ ...prev, [questionId]: optIndex }))
  submitQuizAnswer({ question_id: Number(questionId), user_answer: String(optIndex) })
  //                                                         ^^^^^^^^^^^^^^^^^
  //                                    optIndex 是 0/1/2/3，不是 A/B/C/D
}, [])
```

**后端判分逻辑**（`quiz_helpers.py` `normalize_answer`）：

```python
def normalize_answer(answer, question_type):
    value = answer_to_storage(answer)
    return value.strip().upper()  # "0" → "0", vs correct_answer "A" → "A"
```

结果 `"0" == "A"` → 永远为 `False`。

**修复**：传选项标签而非下标。

```typescript
// practice.tsx
const handleSelectSingle = useCallback((questionId: string, optIndex: number) => {
  const label = currentQuestion.options[optIndex]?.label  // "A", "B", "C", "D"
  setAnswers(prev => ({ ...prev, [questionId]: optIndex }))
  submitQuizAnswer({ question_id: Number(questionId), user_answer: label })
}, [currentQuestion])
```

> 多选题同理：需传标签的逗号分隔字符串 `"A,C,D"` 而非下标的 JSON 数组 `"[0,2,3]"`。

### 3.2 移出错题本：ID 语义偏差

**影响页面**：错题本页 (`pages/quiz/wrong-book`)

**根因**：修复 #2.1 后，`item.id` 将从 record ID 变成 question ID。但 `DELETE /api/quiz/wrong-book/{id}` 的路径参数是 **record ID**。

**当前巧合**：由于 #2.1 的 bug，`item.id` 恰好是 record ID，所以删除能正常工作。修复 #2.1 后，`item.id` 变成 question ID，删除会失败。

**修复**：错题本响应需同时返回 question ID 和 record ID。

```typescript
// quizService.ts — getWrongBook 修复后
return items.map(item => ({
  ...toQuizQuestion(item.question),
  recordId: item.id,        // 保留 record ID 用于删除
  wrongCount: ...,           // 需要从后端补充
  wrongDate: item.updated_at,
}))
```

### 3.3 签到功能：前端无访问入口

**影响**：`pages/quiz/checkin` 页面和签到 API

`pages/quiz/index` 页面中 `QUIZ_INDEX` 路由从未被任何导航引用，导致包含打卡入口的 `CheckinBar` 组件不可达，整个签到功能（`GET/POST /api/quiz/checkin`）成为事实死代码。如确定不需要，可清理前后端签到相关代码。
