# 认证报名流程 — 接口请求流程

> 日期：2026-06-09

---

## 流程图

```
进入认证列表页
  │
  ├── GET /api/cert/certifications             ← 加载认证类型列表
  │     可选 ?vendor=H3C 按厂商筛选
  │
  ▼
点击某个认证 → 进入认证详情页
  │
  ├── GET /api/cert/certifications/{cert_id}   ← 加载该认证详情
  │     返回：价格、vendor、pay_first、requires_xuexin
  │
  ▼
点击报名 → 进入报名表单页
  │
  ├── GET /api/user/profile                    ← 自动填表数据源
  │     id_card_raw → 身份证号框（明文，需 verified）
  │     phone_raw   → 手机号框（明文，需 verified）
  │     real_name   → 姓名框
  │     age         → 年龄框
  │     pinyin      → 拼音
  │
  ├── GET /api/user/identity                   ← 检查认证状态
  │     status=pending  → 提示"认证审核中"
  │     status=rejected → 引导重新认证
  │     不存在          → 引导先去实名认证
  │
  ▼
用户填写/确认表单 → 提交报名 → 创建订单
  │
  ├── POST /api/orders
  │     请求体：cert_type, candidate_name, candidate_phone, candidate_idcard
  │     返回：order_id, status=pending, expires_at
  │
  ▼
进入支付页
  │
  ├── POST /api/payment/prepay  { order_id }
  │     返回 prepay_id, nonce_str, pay_sign, time_stamp …
  │     前端调用 wx.requestPayment 拉起微信支付
  │
  ▼
支付完成
  │
  └── 微信异步回调 → POST /api/payment/callback
        更新 order.status → "paid"
        inventory.available_quota -= 1
```

---

## 接口详解

### 1. `GET /api/cert/certifications` — 认证列表

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| vendor | str | 否 | 按厂商筛选：H3C / 深信服 / NISP / 人社 |

**响应** `list[CertificationResponse]`：

| 字段 | 说明 |
|------|------|
| name | 认证名称（英文）如 `h3c_ne` |
| chinese_name | 认证中文名 |
| code | 认证编码，如 `H3C-NE` |
| vendor | 厂商 |
| requires_xuexin | 是否需要学信网认证 |
| pay_first | 是否先付费后填表 |

---

### 2. `GET /api/cert/certifications/{cert_id}` — 认证详情

继承认证列表全部字段，额外包含：

| 字段 | 说明 |
|------|------|
| created_at | 创建时间 |
| updated_at | 更新时间 |

---

### 3. `GET /api/user/profile` — 表单预填

| 字段 | 返回条件 | 说明 |
|------|---------|------|
| id_card_raw | identity.status = verified | 明文身份证号，供报名表单自动填充 |
| phone_raw | identity.status = verified | 明文手机号 |
| real_name | identity 存在即返回 | 真实姓名 |
| age | identity 存在 + 身份证 18 位即返回 | 从身份证号推算年龄 |
| pinyin | identity 存在即返回 | 姓名拼音 |
| first_name / last_name | identity 存在即返回 | 名/姓拼音 |
| identity_status | identity 存在即返回 | pending / verified / rejected |

---

### 4. `POST /api/orders` — 创建订单

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| cert_type | str | ✅ | 认证类型代码，如 `H3C-NE` |
| candidate_name | str | ✅ | 考生姓名 |
| candidate_phone | str | ✅ | 考生手机号（11 位） |
| candidate_idcard | str | 否 | 考生身份证号（18 位，含校验位） |
| extra_data | dict | 否 | 差异化报名数据 |
| attachments | list[str] | 否 | 上传材料 URL 列表 |

**响应** `OrderResponse`：

| 字段 | 说明 |
|------|------|
| id | 订单 ID |
| status | pending（待支付） |
| price | 订单金额（分） |
| out_trade_no | 商户订单号 |
| expires_at | 订单过期时间（超时未支付自动关闭） |

---

### 5. `POST /api/payment/prepay` — 微信支付预下单

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| order_id | int | ✅ | 订单 ID |

**响应** `PaymentPrepayResponse`：

| 字段 | 说明 |
|------|------|
| prepay_id | 微信支付 prepay_id |
| nonce_str | 随机字符串 |
| time_stamp | 时间戳 |
| package | `prepay_id=xxx` |
| sign_type | 签名类型 |
| pay_sign | 签名 |

前端拿到后调用 `wx.requestPayment()`。

---

### 6. `POST /api/payment/callback` — 微信支付回调

由微信服务器异步 POST，服务端验签后更新订单状态。

---

## 状态流转

```
用户未实名 → 跳转 POST /api/user/identity 提交实名
用户已实名(verified) → 表单自动填 id_card_raw / phone_raw / real_name
用户已实名(pending) → 仍可报名，但无法自动填表

order.status: pending → paid → completed
                           ↘ refunded (退款)
                           ↘ closed   (超时/手动关闭)
```
