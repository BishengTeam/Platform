# 报名缴费页面开发任务表（两人并行版）

> **基于**: `docs/报名缴费页面开发任务表-审计版.md`
> **技术栈**: Taro 4 + React 18 + TypeScript + SCSS Modules + NutUI
> **当前阶段**: 纯前端 UI 开发，数据层全部 mock，不接入后端 API

---

## 一、任务分工概览

| 角色 | 负责页面 | 负责组件/工具 |
|------|----------|---------------|
| **开发者 A** | 报名专区 index + 报名表单 form | FormInput、validator.ts |
| **开发者 B** | 报名确认 confirm + 支付结果 result | PriceRow、EmptyState |

## 二、第 0 轮：共同基础设施（先做，半天）

| # | 任务 | 负责人 | 产出物 |
|---|------|--------|--------|
| C1 | 注册路由 + 扩展 STRINGS | **B** | `app.config.ts` 添加 subPackages；`routes.ts` 添加 ROUTES；`strings.ts` 添加报名/支付文案 |
| C2 | 创建类型定义 | **A** | `src/types/registration.ts` — `Certification`、`RegistrationFormData`、`PaymentResult`；更新 `index.ts` re-export |
| C3 | 创建 mock 数据 + dataService getter | **B** | `src/constants/mock/registration.ts` — 认证列表、mockFormData；更新 mock/index.ts、dataService.ts 添加 getter |
| C4 | 创建表单校验工具 | **A** | `src/utils/validator.ts` — 姓名/手机/身份证正则校验 |

**关键**: A 和 B 在 C1-C4 开始前**先对齐接口**：

- 页面路由路径（`/pages/registration/index`、`/pages/registration/form`、`/pages/registration/confirm`、`/pages/payment/result`）
- URL 参数名约定（`cert_id`、`order_id`）

## 四、第 2 轮：页面联调（A + B 共同）

> 本阶段仅做**页面间数据打通**，不涉及后端 API。

| # | 任务 | 负责人 | 说明 |
|---|------|--------|------|
| L1 | 打通 index → form | **A** | index 卡片点击携带 `cert_id` 跳转 form，form 根据 `cert_id` 从 mock 数据中查找对应认证信息展示 |
| L2 | 打通 form → confirm | **A + B** | A 确认 storage key 一致；B 去掉 mockFormData，改为从 `Taro.getStorageSync` 读取真实表单数据 |
| L3 | 打通 confirm → result | **B** | 模拟支付完成后正确跳转 result，携带 `order_id` + `status` |
| L4 | 全流程走通 | **A + B** | index → form → confirm → 模拟支付 → result → 订单页，完整走通 |
| L5 | 边界状态处理 | **A + B** | 支付失败后重新支付、表单校验失败提示、空列表状态等 |

## 七、B 的 Mock 自测策略

B 开发 confirm 和 result 页时，A 的 index/form 页可能还未完成。B 使用以下方式独立自测：

```typescript
// confirm 页 useLoad 中使用（代替从 storage 读取）
const mockFormData: RegistrationFormData = {
  cert_id: 'h3cne-rs-plus',
  cert_name: 'H3CNE-RS+ 认证工程师',
  real_name: '张三',
  phone: '13800008888',
  id_card: '330106199001011234',
  identity_type: 'enterprise',
  price: 1200,
  coupon_count: 0,
  enterprise_name: '',
}

// result 页 useLoad 中使用（代替从 confirm 传参）
const mockOrderResult = {
  order_id: 'ORD20260508123456',
  status: 'success', // 或 'fail'，切换测试双状态 UI
  amount: 1200,
  cert_name: 'H3CNE-RS+ 认证工程师',
}
```

联调时替换为：
- confirm 页：`Taro.getStorageSync('registration_form_data')`
- confirm 页支付：`setTimeout` 模拟 → `Taro.navigateTo` 跳转 result
- result 页：从 `useLoad(options)` 的 `options` 参数读取 `order_id` 和 `status`

## 八、Storage Key 与 URL 参数约定

| 约定项 | 值 | 说明 |
|--------|-----|------|
| 表单数据 storage key | `registration_form_data` | form 页写入，confirm 页读取 |
| index → form URL 参数 | `?cert_id=xxx` | 传递认证项目 ID |
| form → confirm URL 参数 | 无需参数 | 数据走 storage |
| confirm → result URL 参数 | `?order_id=xxx&status=success\|fail` | 传递订单 ID + 模拟支付结果 |
| result → orders URL 参数 | 无 | 跳转现有订单列表页 |
