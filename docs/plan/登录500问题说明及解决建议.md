# `POST /api/auth/login` 返回 HTTP 500 — 问题说明及解决建议

> 日期：2026-06-10  
> 服务地址：`http://64.90.3.51:10001`  
> 框架：FastAPI (uvicorn)  
> 前端：Taro 微信小程序  

---

## 一、问题现象

微信开发者工具控制台报错：

```
native-apis.js:44 POST http://64.90.3.51:10001/api/auth/login 500 (Internal Server Error)
```

**触发路径**：用户在小程序授权页点击「微信登录」按钮 → `Taro.login()` 获取微信 code → 调 `wxLogin(code)` → 后端返回 HTTP 500。

## 三、后端根因

### 3.1 完整异常堆栈

```text
pydantic_core._pydantic_core.ValidationError: 3 validation errors for UserProfile

id
  Field required [type=missing, input_value={'user_id': 358}, input_type=dict]
openid
  Field required [type=missing, input_value={'user_id': 358}, input_type=dict]
created_at
  Field required [type=missing, input_value={'user_id': 358}, input_type=dict]
```

### 3.2 错误调用链

```
app/api/auth.py:30
    result = await AuthService().login(body.code)
        ↓
app/services/auth.py:39 或 :50
    db.add(UserProfile(user_id=user.id))
        ↓
Pydantic ValidationError → HTTP 500
```

### 3.3 原因：Import 名称遮蔽

**直接原因**：`app/services/auth.py` 中存在 Python import 名称冲突，导致运行时 `UserProfile` 引用的是 Pydantic schema 而非 SQLAlchemy ORM 模型。

**冲突来源**（`app/services/auth.py` 第 9-10 行）：

```python
from app.domain.user.src.index import DeletedOpenid, User, UserProfile   # ① SQLAlchemy ORM 模型
from app.schemas.user import LoginResponse, RefreshResponse, UserProfile  # ② Pydantic schema → 覆盖①
```

第 ② 行的 `UserProfile`（Pydantic schema，`app/schemas/user.py:14`）**覆盖**了第 ① 行的 `UserProfile`（SQLAlchemy ORM 模型，`app/domain/user/src/model/user_profile.py:8`）。

**触发 commit**：`2d2906df`（2026-06-10 05:20，"管理端 PUT /admin/users/{id}/profile 加固"），该 commit 在第 ① 行新增了 `UserProfile` 的导入，与已存在的第 ② 行形成冲突。

### 3.4 两个 UserProfile 的实际结构

| | Domain 层（SQLAlchemy） | Schema 层（Pydantic） |
|---|---|---|
| **路径** | `app/domain/user/src/model/user_profile.py` | `app/schemas/user.py` |
| **继承** | `Base, TimestampMixin` | `BaseModel` |
| **字段** | `user_id`（FK）、`nickname`、`email`、`phone` | `id`（必填）、`openid`（必填）、`nickname`、`phone`、`created_at`（必填） |
| **自增/默认值** | `id` 自增，`created_at` DB 默认 | 无（Pydantic 显式必填） |

**代码期望** `db.add(UserProfile(user_id=user.id))` 使用 Domain 层的 ORM 模型 —— 只需 `user_id` 即可，其余字段由数据库自动填充。

**实际发生**：由于名称被覆盖，`UserProfile(user_id=user.id)` 实际调用的是 Pydantic schema 构造函数，缺少 `id`、`openid`、`created_at` 三个必填字段，立即抛出 `ValidationError`。

### 3.5 受影响代码位置

`app/services/auth.py` 中有两处受影响，一处正常：

| 行号 | 代码 | 场景 | 状态 |
|------|------|------|------|
| 39 | `db.add(UserProfile(user_id=user.id))` | 新用户注册后创建 Profile | ❌ 使用错误的 Pydantic schema |
| 45 | `select(UserProfile).where(...)` | 存量用户查询 Profile | ❌ 使用错误的 Pydantic schema |
| 50 | `db.add(UserProfile(user_id=user.id))` | 存量用户补建 Profile | ❌ 使用错误的 Pydantic schema |
| 65 | `UserProfile.model_validate(user)` | 构造登录响应 | ✅ 正确使用 Pydantic schema |

## 四、修复方案

### 文件：`app/services/auth.py`

**方案：给 Domain 层 `UserProfile` 加别名，消除 import 名称冲突。**

Domain 层 `UserProfile`（SQLAlchemy ORM 模型）本身只需要 `user_id`，`id` 和 `created_at` 由数据库自动填充。不需要补任何字段——只需让代码引用正确的类。

### 4.1 修改 import（第 9 行）

**当前：**

```python
from app.domain.user.src.index import DeletedOpenid, User, UserProfile
```

**修复：**

```python
from app.domain.user.src.index import DeletedOpenid, User, UserProfile as DomainUserProfile
```

### 4.2 修改受影响的 `UserProfile` 引用

以下三处 `UserProfile` 均需替换为 `DomainUserProfile`：

```python
# 新用户分支 — 创建 Profile（第 39 行）
db.add(DomainUserProfile(user_id=user.id))

# 存量用户分支 — 查询 Profile（第 45 行）
select(DomainUserProfile).where(DomainUserProfile.user_id == user.id)

# 存量用户分支 — 补建 Profile（第 50 行）
db.add(DomainUserProfile(user_id=user.id))
```

### 4.3 保持不变的部分

第 65 行的 `UserProfile.model_validate(user)` 本来就该使用 Pydantic schema，**不需要修改**。

### 4.4 修复后代码片段

```python
from app.domain.user.src.index import DeletedOpenid, User, UserProfile as DomainUserProfile
from app.schemas.user import LoginResponse, RefreshResponse, UserProfile

# ...

# 新用户 — 创建 Profile
db.add(DomainUserProfile(user_id=user.id))

# 存量用户 — 查询 Profile
select(DomainUserProfile).where(DomainUserProfile.user_id == user.id)

# 存量用户 — 补建 Profile
db.add(DomainUserProfile(user_id=user.id))

# 响应构造（不变）
user=UserProfile.model_validate(user),
```

### 4.5 为何不需要补字段

| 字段 | Domain 层来源 | 填充方式 |
|------|-------------|---------|
| `id` | `TimestampMixin`（`autoincrement=True`） | 数据库自动生成 |
| `user_id` | 自身定义（FK） | 代码传入 ✅ |
| `nickname` / `email` / `phone` | 自身定义（nullable） | 允许为空 |
| `created_at` | `TimestampMixin`（`server_default=func.now()`） | 数据库自动填充 |
| `updated_at` | `TimestampMixin`（`server_default=func.now()`） | 数据库自动填充 |

Domain 层 `UserProfile` **没有** `openid` 字段——`openid` 属于 `User` 表（`app/domain/user/src/model/user.py`）。

### 4.6 额外确认项

| 检查项 | 说明 |
|--------|------|
| Domain 模型路径 | `app/domain/user/src/model/user_profile.py`，非 `app/models/user.py` |
| `openid` 归属 | 存储在 `User` 表，不在 `UserProfile` 表 |
| 数据库 migration | 确认 `user_profile` 表已通过 Alembic migration 创建 |
| 旧表清理 | `user_identity` 不再用于登录，确认无残留引用 |
